üéØ Brute Force Attack Kya Hai?

"Brute Force Attack" ka topic "API Hacking" roadmap mein "Shurwati Hamle" (Initial Attacks) ke phase mein "Authentication Bypass" section ke andar aata hai.

API security ke hawalay se, brute force attack credentials (jaise usernames aur passwords, ya API keys) ko crack karne ka ek tareeqa hai, jismein systematically har possible combination ko try kiya jata hai jab tak sahi wala mil na jaye.

üîë Simple Misaal

Ye aisa hai jaise aap chabiyon ke guchche mein har chabi ko azmaate hain jab tak woh lock kholne wali chabi na mil jaye. Attacker har possible password try karta rehta hai jab tak correct password nahi mil jata.

üìä Interesting Facts

  • Modern computers 1 second mein millions passwords try kar sakte hain
  • 4-digit PIN (10,000 combinations) seconds mein crack ho sakta hai
  • 8-character password (lowercase only) = 200 billion combinations
  • 80% brute force attacks 6 characters ya kam ke passwords target karte hain
  • Rate limiting na hone par ek API 24 hours mein millions attempts face kar sakti hai

‚öôÔ∏èBrute Force Attack Kaise Kaam Karta Hai?

Automated Guessing

Attackers automated tools ka istemal karte hain taake tezi se credentials ke liye bohot badi tadad mein guesses bhejein. Ye tools har second mein hazaron ya lakhon combinations try kar sakte hain.

  • Scripts aur programs automatically run hote hain
  • Parallel processing se speed increase hoti hai
  • Wordlists aur algorithms ka combination
  • 24/7 operation possible hai

Process Flow

  1. Target Selection: API endpoint identify karna (/api/login)
  2. Wordlist Preparation: Possible passwords ki list tayar karna
  3. Request Generation: Har password ke liye HTTP request banana
  4. Response Analysis: Success ya failure determine karna
  5. Iteration: Successful hone tak continue karna

üìãBrute Force Attack Ki Types

1. Dictionary Attacks

Ye brute force ki ek qism hai jismein attacker aam istemal hone wale usernames aur passwords ki pehle se tayyar shuda list ka istemal karta hai.

  • Common passwords ki pre-compiled lists
  • Leaked password databases
  • Language-specific wordlists
  • Industry-specific terms

üí° Common Passwords Dictionary:

password password123 admin letmein welcome qwerty123 monkey

2. Credential Stuffing

Ismein doosre breaches se leaked username aur password pairs ka istemal karna shamil hai, ye soch kar ke users aksar apni credentials mukhtalif services par dobara istemal karte hain.

  • Previous data breaches ka data use karna
  • Email:Password combinations
  • High success rate (5-10%)
  • Multiple APIs par simultaneously test karna

üí° Process:

1. LinkedIn breach se 10M credentials leak

2. Attacker in credentials ko banking API par try karta hai

3. Password reuse ki wajah se 500K accounts compromise

3. Hybrid Attacks

Dictionary attacks ko numerical ya special character variations ke sath milana.

  • Dictionary words + numbers: password123
  • Dictionary words + special chars: password!
  • Capitalization variations: Password, PASSWORD
  • Leet speak: p@ssw0rd

üí° Variations Example:

summer ‚Üí summer2023, Summer123, summer!, summ3r

4. Pure Brute Force

Har mumkin character combination ko try karna, jo behtar waqt talab ho sakta hai lekin agar koi doosre defenses maujood na hon toh sahi credential milne ki zamanat deta hai.

  • All possible combinations systematically
  • Start from simple (a, aa, aaa...)
  • Move to complex combinations
  • Time-consuming but guaranteed success (eventually)

üí° Time Estimates (for offline cracking):

4 chars (lowercase): 0.02 seconds

6 chars (lowercase): 5 minutes

8 chars (alphanumeric): 3 weeks

12 chars (mixed): Centuries

5. Reverse Brute Force

Ek common password choose karein aur use multiple usernames ke saath try karein.

  • Common password select karna: password123
  • Large username list ke saath try karna
  • Account lockout bypass karne mein useful
  • Large organizations mein effective

üéØBrute Force Attack Ke Targets

Bunyadi tor par brute force attacks ye targets karte hain:

1. Usernames Aur Passwords

Attackers API user accounts ke liye valid username/password pairs guess karne ki koshish karte hain.

üí° Common Scenarios:

POST /api/login POST /api/auth/signin POST /api/v1/authenticate

Body: {"username":"admin","password":"guess123"}

2. API Keys

Agar koi API authentication ke liye keys ka istemal karti hai, toh attackers valid API keys guess karne ki koshish kar sakte hain.

üí° API Key Patterns:

X-API-Key: XXXX-XXXX-XXXX-XXXX Authorization: Bearer eyJhbGc...

Agar key format predictable hai to brute force possible hai

3. OTP (One-Time Passwords)

Traditional brute force ke liye kam common hai, lekin attackers OTPs guess karne ki koshish kar sakte hain agar unke generation ya validation mein kamzorian hon.

  • 4-digit PIN: 10,000 combinations (asaani se try ho sakte hain)
  • 6-digit OTP: 1 million combinations (time limit mein mushkil)
  • Time-based OTPs zyada secure hain
  • SMS OTPs interception ka risk

4. Session Tokens

Agar session tokens predictable pattern follow karte hain to unhe brute force kiya ja sakta hai.

üí° Vulnerable Token Examples:

session_id=12345 (sequential) token=userid_timestamp (predictable)

üîßAPIs Ke Saath Ye Kaise Kaam Karta Hai

Attackers aam tor par ek API ke authentication endpoint par mukhtalif credential combinations ke sath baar baar HTTP requests bhejte hain, jab tak ek successful authentication response na mil jaye.

üí° Typical Attack Flow:

1. POST /api/login {"user":"admin","pass":"test1"} ‚Üí 401 Unauthorized 2. POST /api/login {"user":"admin","pass":"test2"} ‚Üí 401 Unauthorized 3. POST /api/login {"user":"admin","pass":"test3"} ‚Üí 401 Unauthorized ... 1000. POST /api/login {"user":"admin","pass":"admin123"} ‚Üí 200 OK

Success! Credentials mil gaye

‚ö†Ô∏èAPIs Ke Liye Brute Force Attacks Kyun Khatra Hain?

1. Weak Credentials

Agar koi API users ko kamzor ya aasani se guess hone wale passwords set karne deti hai, toh woh bohot zyada vulnerable ho jati hai. Dictionary attack se easily crack ho sakti hai.

2. Lack of Rate Limiting

Woh APIs jo ek khaas time frame mein ek hi IP address ya user account se na-kaam login attempts ki tadad ko mahdood nahin karte, woh prime targets hote hain. Unlimited attempts allow hone se brute force asaan ho jata hai.

3. No Account Lockout

Agar kai na-kaam attempts ke baad account temporarily lock nahin hota, toh ek attacker be-rok tok koshish karta reh sakta hai. 24/7 attacks possible hain.

4. Verbose Error Messages

Aisi error messages jo invalid username aur invalid password ke darmiyan farq batati hain, attackers ko apni guesses ko kam karne mein madad kar sakti hain.

‚ùå Bad Practice:

"Username does not exist" "Password is incorrect"

‚úÖ Good Practice:

"Invalid credentials"
5. Predictable Patterns

Sequential user IDs, predictable API keys, ya default username patterns brute force ko aur asaan banate hain.

üõ†Ô∏èBrute Force Attack Ke Tools

1. Hydra

Sabse popular aur powerful network authentication cracker. Multiple protocols support karta hai.

hydra -l admin -P passwords.txt https://api.example.com/login

Features: Fast, multi-threaded, supports 50+ protocols

2. Burp Suite Intruder

Web application testing ke liye best tool. Custom positions aur payloads ke saath flexible attack types.

Attack Types: Sniper, Battering ram, Pitchfork, Cluster bomb

3. Medusa

Fast, parallel password cracker. Modular design ke saath multiple services support.

medusa -h api.example.com -u admin -P wordlist.txt -M http

4. Patator

Python-based multi-purpose brute-forcer. Flexible aur extensible.

patator http_fuzz url=https://api.example.com/login method=POST body='user=admin&pass=FILE0' 0=passwords.txt

5. Custom Python Scripts

requests library ke saath custom brute force scripts likhi ja sakti hain for specific scenarios.

import requests for password in wordlist: response = requests.post(url, json={"user":"admin","pass":password}) if response.status_code == 200: print(f"Found: {password}")

6. Hashcat

Offline password cracking ke liye. Agar password hashes leak ho gayen hon to use kiya jata hai.

hashcat -m 0 -a 0 hashes.txt wordlist.txt

üõ°Ô∏èBrute Force Attacks Se Bachao

In defense mechanisms ko implement kar ke brute force attacks se bachaya ja sakta hai:

1. Strong Password Policies

Complex passwords ko enforce karein jo brute force se crack karna mushkil hon.

  • Minimum 12 characters length
  • Uppercase, lowercase, numbers, special characters required
  • No dictionary words
  • No common patterns (123456, qwerty)
  • Regular password changes
2. Rate Limiting

Ek khaas waqt mein login attempts ki tadad ko mahdood karein.

  • Per IP: Max 5 attempts per minute
  • Per user account: Max 5 attempts per hour
  • Global rate limit: Total requests per second
  • Exponential backoff implement karen
3. Account Lockout

Chand na-kaam attempts ke baad accounts ko temporarily lock kar dein.

  • 5 failed attempts ‚Üí 15 minutes lockout
  • 10 failed attempts ‚Üí 1 hour lockout
  • 20 failed attempts ‚Üí Account suspended (manual unlock)
  • Email notification bhejein user ko
4. CAPTCHAs

Human users aur automated bots mein farq karne ke liye CAPTCHAs lagayein.

  • 3 failed attempts ke baad CAPTCHA show karen
  • reCAPTCHA v3 for invisible protection
  • hCaptcha as alternative
  • Custom challenges implement karen
5. Multi-Factor Authentication (MFA)

Password ke ilawa security ki ek aur layer shamil karein.

  • SMS/Email OTP
  • Authenticator apps (Google Authenticator, Authy)
  • Hardware tokens (YubiKey)
  • Biometric authentication
6. IP Blacklisting

Suspicious activity dikhane wale IP addresses ko block karein.

  • Automatic blacklisting after threshold
  • Known malicious IPs ki lists use karen
  • Geo-blocking implement karen (agar applicable ho)
  • Temporary vs permanent bans
7. Monitoring Aur Alerting

Real-time monitoring aur alerts setup karen.

  • Failed login patterns track karen
  • Unusual activity detect karen
  • Security team ko alerts bhejein
  • Logs analyze karen regularly
8. Progressive Delays

Har failed attempt ke baad delay increase karen.

  • 1st fail: No delay
  • 2nd fail: 2 seconds delay
  • 3rd fail: 5 seconds delay
  • 4th fail: 10 seconds delay
  • Subsequent: Exponentially increase

‚ö†Ô∏è Testing Guidelines

Brute force attacks sirf authorized testing environments par karein. Production systems par unauthorized brute force testing illegal hai aur serious legal consequences ho sakte hain. Hamesha bug bounty programs ya proper authorization ke saath testing karein.

üí° Ahem Nuqta

Brute force attacks ko samajhna API hackers ke liye bunyadi hai kyunki ye API resources tak unauthorized access haasil karne ka ek aam ibtedai tareeqa hai. Proper defenses implement kar ke organizations apni APIs ko is common attack se bacha sakte hain.

üìå Khulasa

Brute Force Attacks systematically har possible combination ko try karne ka method hai - simple lekin effective. Modern computing power ki wajah se ye attacks bohot tezi se execute ho sakte hain, khaas kar jab koi proper defenses na hon.

Successful defense ke liye multiple layers ki zaroorat hoti hai: strong passwords, rate limiting, account lockouts, CAPTCHAs, aur MFA. Ek bhi layer weak hone se attacker success ho sakta hai.

Developers: Apni APIs mein comprehensive brute force protection implement karen. Users: Strong, unique passwords istemal karen aur MFA enable karen jahan available ho. Security Teams: Regular monitoring aur testing karte rahein.
