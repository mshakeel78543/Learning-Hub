üéØ API Key Kya Hai?

Sab se pehle, yeh samajhna zaroori hai ke API key hoti kya hai. API key ek munfarid (unique) pehchan hoti hai jo kisi user, developer, ya call karne wale program ko API tak rasai (access) dene ke liye istemal hoti hai.

Yeh darasal ek raazdar token hota hai jo khaas API functionalities aur data tak pahunchne ki ijazat deta hai.

üé≠ Simple Misaal

Aap isay ek password ya application ke liye ek key card samajh sakte hain. Jaise ek building mein entry ke liye aapko key card chahiye, waise hi API access ke liye API key chahiye hoti hai.

Common API Key Formats:

X-API-Key: sk_live_51H7Xj2eZvKYlo2C...

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

api_key=AIzaSyD-9tSrke72PouQMnMX-a7eZSW0jkFMBWY

"API Keys Compromise" topic "API Hacking" roadmap ke "Shurwati Hamle" (Initial Attacks) section mein "Authentication Bypass" ke andar aata hai.

‚ùì"API Keys Compromise" Ka Kya Matlab Hai?

"API Keys Compromise" ka matlab hai jab koi ghair-ikhtiyar (unauthorized) teesra fard kisi jaiz (legitimate) API key tak rasai haasil kar leta hai.

Jab ek hacker ko API key mil jaati hai, to woh jaiz user ya application ka bhes badal kar API ko ghair-ikhtiyar (unauthorized) requests bhej sakta hai.

üîìAPI Keys Compromise Kaise Hoti Hain?

API keys mukhtalif tareeqon se compromise ho sakti hain:

1. Code/Applications Mein Hardcode Karna

Developers kabhi kabhi API keys ko directly client-side code mein daal dete hain. Is se hamlawaron ke liye code ka muaina kar ke unhe dhoondhna aasan ho jata hai.

üí° Hardcoded Keys Examples:

// JavaScript mein (Frontend) const API_KEY = "sk_live_51H7Xj2eZvKYlo2C..."; fetch('https://api.example.com/data', { headers: {'X-API-Key': API_KEY} });

Ya

// Mobile App Code mein String apiKey = "AIzaSyD-9tSrke72PouQMnMX...";
  • JavaScript files mein plaintext keys
  • Mobile app binaries mein embedded keys
  • Configuration files committed to codebase
  • HTML source code mein hidden fields
  • React/Angular app builds mein exposed keys

2. Version Control Systems (Git, SVN)

Galti se API keys ko public ya ghair-mehfooz private repositories mein commit karna ek bohot aam galti hai.

üí° Git History Mein Keys:

git log --all -p | grep -i "api_key" git log --all -p | grep -i "secret"

Ya

// .env file accidentally committed API_KEY=sk_live_51H7Xj2eZvKYlo2C... DB_PASSWORD=mypassword123
  • GitHub/GitLab public repositories
  • Git history mein purani commits
  • .env files accidentally committed
  • config.json, secrets.yaml files
  • Forked repositories mein keys
  • GitHub Gists mein shared code

3. Ghair-mehfooz Configuration Files

API keys ko server par aasaan pahunch wale plain-text configuration files mein store karna.

üí° Exposed Config Files:

https://example.com/.env https://example.com/config.json https://example.com/application.properties https://example.com/wp-config.php.bak
  • Publicly accessible .env files
  • Backup configuration files (.bak, .old)
  • Debug/test configuration files
  • Docker environment files
  • Kubernetes secrets improperly configured

4. Man-in-the-Middle Attacks

Agar API requests ko encrypt nahin kiya gaya hai (masalan, HTTPS ki bajaye HTTP istemal karna), to ek hamlawar traffic ko rok kar API key chura sakta hai.

üí° MITM Scenario:

// Insecure HTTP request GET http://api.example.com/data X-API-Key: sk_live_51H7Xj2eZvKYlo2C...

Attacker network par sniffing kar ke key dekh sakta hai

  • HTTP (non-encrypted) connections
  • Public Wi-Fi networks par sniffing
  • ARP spoofing attacks
  • DNS hijacking
  • SSL stripping attacks

5. Kamzor Access Controls

Agar API key rakhne wali file ya directory ke permissions kamzor hon, to server par ghair-ikhtiyar users us tak rasai haasil kar sakte hain.

üí° Weak Permissions:

-rw-r--r-- 1 root root .env (World readable!) chmod 644 /var/www/.env

Better:

-rw------- 1 user user .env (Owner only) chmod 600 /var/www/.env
  • World-readable configuration files
  • Weak file system permissions
  • Shared hosting environments
  • Container escape vulnerabilities
  • Local privilege escalation

6. Social Engineering/Phishing

Kisi developer ya administrator ko dhoka de kar API key zaahir karwa lena.

üí° Phishing Examples:

Email: "Your API key is expiring. Please verify by clicking here..."

Fake support call: "We need your API key for troubleshooting..."

Malicious Slack/Discord messages

  • Targeted phishing emails
  • Fake support calls
  • Impersonation attacks
  • Compromised Slack/Teams accounts
  • Fake API documentation sites

7. Brute-Forcing/Guessing

Agarchay complex keys ke liye yeh kam aam hai, simple ya behtar tareeqe se na banai gai API keys ka andaza lagaya ja sakta hai.

üí° Weak Key Patterns:

api_key=123456 api_key=companyname_001 api_key=user_12345

Predictable patterns easily guessed

  • Short key lengths
  • Sequential patterns
  • Dictionary-based keys
  • Timestamp-based generation
  • Weak randomness

8. Data Breaches

Agar API keys store karne wala system breach ho jaye, to keys churai ja sakti hain.

üí° Breach Scenarios:

Database compromise se API keys table leak

Backup files accidentally exposed

Third-party service breach

Cloud storage misconfiguration (S3 buckets)

  • Database breaches
  • S3 bucket misconfigurations
  • Backup file leaks
  • Third-party vendor compromises
  • Ransomware attacks

9. Documentation Mein Zaahir Hona

Kabhi kabhi, API keys (khaas kar misali keys) galti se public documentation mein reh sakti hain.

üí° Documentation Exposure:

// API documentation example: curl -H "X-API-Key: sk_live_actual_key" \ https://api.example.com/users

Developers forget to replace with placeholder

  • Public API documentation
  • Blog posts aur tutorials
  • Stack Overflow answers
  • YouTube video descriptions
  • README files
  • Wiki pages

10. Browser Extensions & Malware

Malicious browser extensions ya malware API keys ko steal kar sakte hain.

üí° Malware Theft:

// Malicious extension chrome.storage.local.get(['api_key'], function(result) { sendToAttacker(result.api_key); });
  • Malicious browser extensions
  • Keyloggers
  • Screen capture malware
  • Clipboard hijacking
  • Memory scraping

üí•API Key Compromise Ke Nuqsanat

Jab API key compromise ho jaye to yeh serious consequences ho sakte hain:

1. Ghair-ikhtiyar Data Tak Rasai

Hamlawar (attacker) us sensitive data ko padh sakta hai, tabdeel kar sakta hai ya mita sakta hai jis tak us API key ko rasai haasil hoti hai.

  • Customer data theft
  • Financial information exposure
  • Personal identifiable information (PII) leaks
  • Business secrets aur intellectual property
  • Database dumping
2. Service Ka Ghalat Istemal

Woh API resources ko istemal kar sakta hai, jis se jaiz malik (legitimate owner) ke liye kharche (costs) peda ho sakte hain.

  • API quotas ka misuse
  • Billing overages (AWS, Google Cloud)
  • Expensive API operations (AI, ML services)
  • Cryptocurrency mining via cloud APIs
  • Spam aur mass operations
3. Privilege Escalation

Agar compromise ki gai API key ko zyada permissions hasil hon, to hamlawar mazeed vulnerabilities talaash kar sakta hai ya zyada sensitive systems tak pahunch sakta hai.

  • Admin-level API keys se complete control
  • Infrastructure access
  • Database credentials exposure
  • Internal systems access
  • Lateral movement possible
4. Denial of Service (DoS)

Ek hamlawar jan boojh kar API key ka ghalat istemal kar ke service ko requests se bhar sakta hai, jis se woh jaiz users ke liye dastiyab nahin rahegi.

  • Rate limit exhaustion
  • Service degradation
  • Account suspension
  • Business operations disruption
5. Reputation Damage

Company ki reputation aur customer trust ka loss.

  • Media coverage of breach
  • Customer confidence loss
  • Partner relationships damage
  • Stock price impact
6. Legal & Compliance Issues

Regulatory violations aur legal consequences.

  • GDPR violations
  • PCI-DSS compliance breaches
  • HIPAA violations
  • Lawsuits aur fines

üö® Real-World Impact

API key compromise ki wajah se kai major incidents hui hain:

  • Uber (2016): GitHub repository se AWS keys leak, 57 million records compromised
  • Toyota (2022): T-Connect API keys exposed for 10 years, 300K+ customers affected
  • Travis CI (2022): API tokens leak, thousands of organizations impacted
  • Crypto wallets: API keys leak se millions of dollars theft

üõ°Ô∏èRok Tham (Mitigation/Defenses)

API key compromise ko rokne ke liye, behtareen tareeqon par amal karna zaroori hai:

1. Environment Variables Ka Istemal

API keys ko servers par environment variables ke taur par store karen, code mein directly nahin.

  • Use .env files (never commit to git)
  • Load environment variables at runtime
  • Keep .env in .gitignore
  • Use different keys for dev/staging/production

üí° Proper Implementation:

// .env file (not in git) API_KEY=sk_live_51H7Xj2eZvKYlo2C... DB_PASSWORD=securepassword

Code mein:

const apiKey = process.env.API_KEY;
2. Makhsoos Secret Management Tools

API keys ko mehfooz tareeqe se store aur retrieve karne ke liye dedicated tools istemal karen.

  • HashiCorp Vault - Enterprise secret management
  • AWS Secrets Manager - Cloud-native solution
  • Azure Key Vault - Microsoft Azure
  • Google Cloud Secret Manager - GCP solution
  • Doppler - Modern secrets management
  • 1Password, LastPass - Team password managers
3. Kabhi Hardcode Na Karen

Front-end code ya aam dastiyab client-side applications mein API keys ko directly embed karne se bilkul parhez karen.

  • Never put keys in JavaScript files
  • Don't embed in mobile app binaries
  • Use backend proxy for sensitive operations
  • Implement proper authentication flow
  • Use short-lived tokens instead
4. Least Privilege Principle

API keys ko sirf utni hi permissions den jitni unke kaam ke liye zaroori hon.

  • Create separate keys for different services
  • Limit scope aur permissions
  • Read-only keys jahan possible ho
  • Resource-specific restrictions
  • Time-based access limitations
5. IP Whitelisting

Agar mumkin ho, to API key ke istemal ko khaas IP addresses ya ranges tak mehdood kar den.

  • Configure allowed IP addresses
  • Use CIDR notation for ranges
  • Combine with VPN for remote access
  • Monitor unauthorized IP attempts
  • Update whitelist regularly
6. Rate Limiting

API endpoints par rate limiting implement karen taake agar key compromise ho bhi jaye, to bhi uska zyada istemal roka ja sake.

  • Per-key rate limits
  • Per-IP rate limits
  • Burst protection
  • Exponential backoff
  • CAPTCHA after threshold
7. Regular Rotation

Waqtan-fawaqtan (periodically) API keys ko rotate karte rahen taake compromise hone wali key ke istemal ka waqfa kam kiya ja sake.

  • Quarterly or monthly rotation
  • Automated rotation workflows
  • Grace period for transitions
  • Immediate rotation on suspicion
  • Track key versions
8. HTTPS Enforce Karen

Hamesha HTTPS istemal karen taake API communication ko encrypt kiya ja sake aur keys ko intercept hone se roka ja sake.

  • TLS 1.2 or higher
  • Valid SSL certificates
  • HSTS headers
  • Certificate pinning (mobile apps)
  • Reject HTTP connections
9. Monitoring Aur Logging

API key ke istemal ko ghair-mamooli sar-garmi ke liye monitor karen aur tamam API calls ko log karen.

  • Log all API key usage
  • Anomaly detection systems
  • Alert on suspicious patterns
  • Geographic anomalies
  • Usage spike detection
  • Failed authentication attempts
10. Automated Scanning

Code repositories aur deployments ko automatically scan karen for exposed keys.

  • git-secrets - Prevent commits
  • truffleHog - Scan git history
  • GitGuardian - Real-time monitoring
  • GitHub Secret Scanning - Built-in protection
  • Pre-commit hooks

üîçTesting For Exposed API Keys

Test 1: Source Code Review

Client-side code mein hardcoded keys dhoondhein.

  1. Browser DevTools open karen
  2. Sources tab mein saari JS files dekhen
  3. Search karen: api_key, apikey, secret, token
  4. Network tab mein requests check karen
  5. Headers aur query parameters analyze karen

Test 2: Git Repository Scanning

Git history mein exposed keys search karen.

  1. Clone repository locally
  2. Run: git log --all -p | grep -i "api_key"
  3. Use truffleHog: trufflehog git https://github.com/user/repo
  4. Check .git folder if exposed publicly
  5. Review commit messages for keywords

Test 3: Configuration Files

Common configuration files access karne ki koshish karen.

  1. Try: https://target.com/.env
  2. Try: https://target.com/config.json
  3. Try: https://target.com/.git/config
  4. Directory fuzzing tools use karen
  5. Backup files check karen (.bak, .old)

Test 4: Mobile App Decompilation

Mobile apps se API keys extract karen.

  1. APK download karen (Android)
  2. Use JADX ya apktool for decompilation
  3. Search for API keys in code
  4. Check strings.xml aur config files
  5. iOS: IPA file extract karen

Test 5: Third-Party Services

Public platforms par exposed keys search karen.

  1. GitHub search: org:company "api_key"
  2. Pastebin searches
  3. Stack Overflow answers
  4. Public Gists
  5. Documentation sites

‚ö†Ô∏è Ethical Testing Warning

Discovered API keys ka misuse karna illegal hai. Agar aapko kisi organization ki exposed API keys milti hain to immediately unhe responsible disclosure ke zariye inform karen. Unauthorized access crime hai aur serious legal consequences ho sakte hain.

üí° Ahem Nuqta

API Key Compromise ek bohot bari vulnerability hai kyunki yeh aapki API ki functionalities aur data tak ghair-ikhtiyar rasai ka baais ban sakti hai. API keys ki mehfooz handling aur management API security ke liye bunyadi ahmiyat rakhti hai. Prevention se behtar koi cure nahi - proper practices ab implement karen taake baad mein regret na karna pade.

üìå Khulasa

API Keys Compromise modern application security mein ek critical threat hai. API keys basically aapki application ki master keys hain - agar yeh galat hathon mein chali jayen to complete system compromise ho sakta hai.

Developers ko chahiye ke wo API keys ko treat karen jaise nuclear codes - extreme care ke saath. Kabhi hardcode na karen, hamesha encrypted aur secure storage use karen, regular rotation implement karen, aur comprehensive monitoring setup karen.

Yaad rakhein: "Prevention is better than cure" - API keys ko secure karna ek continuous process hai, one-time task nahi. Regular audits, automated scanning, aur security awareness training zaroori hain.
