üéØ Google Dorking Kya Hai?

"Google Dorking" (jise Google Hacking bhi kehte hain) ek aisi technique hai jismein Google (ya doosre search engines) ke advanced operators aur search queries ko istemal karte hue khaas, aksar sensitive, maloomat (information) ko talaash kiya jata hai jo websites par publicly maujood ho.

API Hacking ke context mein, yeh ek bohot hi asardar reconnaissance (maloomat jama karna) method hai jise target APIs, web servers, aur in se mutaliq infrastructure ke baare mein jaankari jama karne ke liye istemal kiya jata hai.

‚öôÔ∏èYeh Kaise Kaam Karta Hai?

Google ka search engine bohot badi miqdar mein maloomat ko index karta hai. Khaas search operators ka istemal karke, aap apni search results ko narrow down kar sakte hain taake khaas qism ki files, exposed directories, configuration files, error messages, ya yahan tak ke API keys aur credentials ko dhoond saken jo developers ne ghalti se public chor diye hon.

üõ†Ô∏èAam Google Dorking Operators

Yahan kuch zaroori operators hain jo Google Dorking mein istemal hote hain:

1. site:

Aapki search ko ek khaas domain tak mehdood karta hai.

üí° Misaal:

site:example.com "admin panel"

Sirf example.com par "admin panel" talaash karega

2. inurl:

Aise pages dhundta hai jahan ek khaas lafz ya phrase URL mein maujood ho.

üí° Misaal:

inurl:admin login.php

Aise URLs dhundhega jin mein "admin" aur "login.php" ho

3. intitle:

Aise pages dhundta hai jahan ek khaas lafz ya phrase page ke title mein maujood ho.

üí° Misaal:

intitle:"index of" config

Directory listings ko dekhega jinke title mein "config" ho

4. filetype:

Khaas qism ki files (jaise pdf, doc, xls, txt) ko talaash karta hai.

üí° Misaal:

filetype:env DB_PASSWORD

.env files dhundhega jin mein database passwords ho sakte hain

5. cache:

Ek page ka cached version dikhata hai.

üí° Misaal:

cache:example.com

6. link:

Aise pages dhundta hai jo ek khaas URL se link karte hain.

üí° Misaal:

link:example.com

7. - (minus sign)

Search results se ek lafz ko nikal deta hai.

üí° Misaal:

site:example.com "password" -forum

example.com par "password" dhundhega lekin forum ke results nikal dega

8. OR

Search queries ko combine karta hai (aise pages dhundta hai jin mein dono mein se koi ek lafz ho).

üí° Misaal:

site:example.com (API_KEY OR API_SECRET)

9. "" (quotation marks)

Ek exact phrase ko talaash karta hai.

üí° Misaal:

site:example.com "This is a secret key"

‚ö°API Hacking Ke Liye Yeh Kyun Zaroori Hai?

API hacking ke "Maloomat Jama Karna" (Information Gathering) marhale mein, Google Dorking aapko yeh cheezein uncover karne mein madad karta hai:

1. Exposed API Keys Aur Credentials

Developers kabhi-kabhi API keys ya sensitive credentials ko public accessible files ya code mein hardcode kar dete hain, jinhein search engines index kar sakte hain.

üîç Dork Examples:

site:target.com "X-API-Key" site:target.com "Authorization: Bearer" site:target.com "api_key" OR "apikey" site:github.com target.com "API_KEY"

2. API Documentation

Aap public ya internal API documentation dhoond sakte hain jo publically expose hone ke liye nahi thi, aur is se endpoints, parameters, aur authentication methods ka inkishaf ho sakta hai.

üîç Dork Examples:

site:target.com filetype:pdf "api documentation" site:target.com inurl:swagger site:target.com inurl:api-docs site:target.com "openapi.json"

3. Error Messages Aur Debug Information

Tafseeli error messages kabhi-kabhi internal system information, database structures, ya application logic ko expose kar sakte hain.

üîç Dork Examples:

site:target.com "stack trace" site:target.com "SQL syntax error" site:target.com "Warning: mysql_" site:target.com "Fatal error"

4. Configuration Files

Configuration files mein sensitive settings, database connection strings, ya internal network details ho sakte hain.

üîç Dork Examples:

site:target.com filetype:env site:target.com inurl:config.php site:target.com filetype:xml "database" site:target.com ".env" DB_PASSWORD

5. Directory Listings

Open directory listings development files, backup files, ya doosri sensitive maloomat ko zaahir kar sakte hain jo publically accessible nahi honi chahiye.

üîç Dork Examples:

site:target.com intitle:"index of" /dev site:target.com intitle:"index of" /backup site:target.com intitle:"index of" /admin site:target.com intitle:"index of" .git

6. Subdomains

Jabke yeh direct dorking nahi hai, Google subdomains ko uncover karne mein madad kar sakta hai jo mukhtalif API versions ya mutaliqa services host kar sakte hain.

üîç Dork Examples:

site:*.target.com -www site:*.target.com "api" site:*.target.com inurl:dev

üí°Advanced Tips Aur Combinations

  • Multiple Operators Combine Karein

    site:target.com filetype:pdf inurl:api "credentials"

  • GitHub Par Secrets Dhoondhein

    site:github.com "target.com" "password" OR "api_key"

  • Pastebin Ya Paste Sites Check Karein

    site:pastebin.com "target.com" password

  • Development/Staging Environments

    site:target.com (inurl:dev OR inurl:staging OR inurl:test)

  • Backup Files Dhoondhein

    site:target.com ext:bak OR ext:old OR ext:backup

üí° Pro Tip

Google Dorking ko aur effective banane ke liye, mukhtalif operators ko combine karein aur creative queries banayein. Kabhi kabhi thodi si creativity se bohot valuable information mil sakti hai.

‚ö†Ô∏èKhatrat Aur Akhlaqi Iqdar

Jabke Google Dorking penetration testers aur ethical hackers ke liye ek taqatwar tool hai, yeh yaad rakhna zaroori hai:

1. Qanooniyat (Legality)

Search engines ke zariye publicly available maloomat tak rasai hasil karna aam taur par ghair-qanooni nahi hai. Taham, us maloomat ka ghair-majazi rasai hasil karne ya vulnerabilities ko exploit karne ke liye istemal karna ghair-qanooni hai.

2. Ethical Hacking

Hamesha yaqeeni banayen ke aapke paas kisi bhi system par vulnerabilities ko identify ya exploit karne ki saaf ijazat (maslan, bug bounty program ya penetration test ke liye client se) mojood ho.

3. Responsible Disclosure

Agar aap koi vulnerability ya sensitive information discover karte hain, to responsible disclosure ka tareeqa follow karein. Organization ko pehle inform karein before public disclosure.

‚ö†Ô∏è Tanbih (Warning)

Google Dorking sirf authorized testing ke liye istemal karein. Unauthorized access ya malicious purposes ke liye istemal karna illegal hai aur serious legal consequences ho sakte hain.

üìå Khulasa

Google Dorking API hacking ke ibtedai reconnaissance marhale mein ek bohot ahem qadam hai. Search queries ko maharat se banakar, aap qeemti clues aur mumkina kamzoriyon ko uncover kar sakte hain jo foran zaahir nahi hotin.

Yeh technique aapko exposed API keys, sensitive documentation, error messages, configuration files, aur doosri valuable information dhoondhne mein madad karti hai. Lekin hamesha yaad rakhein ke is tool ka istemal ethical aur legal boundaries ke andar hi karna hai.

Yaad rakhein: Information gathering ka maqsad security ko behtar banana hai, na ke nuksaan pohanchana. Hamesha responsible aur ethical approach follow karein.
