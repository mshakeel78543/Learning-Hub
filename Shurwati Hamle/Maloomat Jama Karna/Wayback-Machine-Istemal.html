üéØ Wayback Machine Kya Hai?

Wayback Machine, jise Internet Archive ne banaya hai, World Wide Web ka ek digital archive hai. Yeh aapko yeh dekhne ki ijazat deta hai ke websites (aur unki bunyadi files) waqt ke saath kaise badli hain.

"Wayback Machine Istemal Karna" (Using the Wayback Machine) API hacking ke "Maloomat Jama Karna" (Information Gathering) marhale mein istemal hone wala ek tareeqa hai.

Ek API hacker ke liye, yeh reconnaissance ke liye ek qeemti tool hai kyunke yeh waqt mein peechey ja kar target ki web presence ke bare mein tareekhi data jama karne mein madad karta hai.

Website: https://web.archive.org

üíéAPI Hacking Ke Liye Kaise Faidamand Hai?

1. Purane Endpoints Aur Documentation Dhundhna

APIs, websites ki tarah, waqt ke saath evolve hote hain. Purane versions mein aisi kamzoriyan ho sakti hain jo naye versions mein theek kar di gai hon. Wayback Machine ek website ke maazi ke snapshots faraham kar sakti hai, jis se yeh cheezein saamne aati hain:

  • Deprecated API endpoints: Aise endpoints jo ab active taur par istemal ya document nahi hote lekin phir bhi live aur kamzor ho sakte hain.
  • Pichli API documentation: Documentation ke purane versions mein endpoints, parameters, ya authentication methods ke bare mein tafseelat ho sakti hain jo ab publically available nahi hain lekin phir bhi exploit ki ja sakti hain.
  • Chhupe hue ya bhool gaye resources: Files, directories, ya API paths jo kabhi public the lekin baad mein hata diye gaye ya restricted kar diye gaye.

2. Sensitive Information Talaash Karna

Kabhi kabhi, sensitive maloomat jaise API keys, developer comments, ya internal configurations galti se website ke purane version ya uski mutaliqa files mein expose ho gaye hon, aur Wayback Machine ne use archive kar liya ho.

üí° Misaal:

Purani JavaScript files mein hardcoded API keys:

const API_KEY = "sk-123456789abcdef";

Purani config files mein database credentials:

db_password: "admin123"

3. API Evolution Ko Samajhna

API ki documentation ya mutaliqa web content mein hone wali tabdeeliyon ko dekh kar, ek hamlawar uske development patterns, ghaflat ke mumkina ilaqon, ya waqt ke saath developers ki taraf se ki gai aam ghaltiyon ke bare mein jankari haasil kar sakta hai.

üõ†Ô∏èAPI Hacking Mein Iska Istemal Kaise Hota Hai?

Ek hamlawar aam taur par Wayback Machine ki website par jata hai aur apne target ka domain enter karta hai. Phir woh website ke mukhtalif archived versions ko dekhta hai.

üìã Basic Steps:

  1. https://web.archive.org par jayein
  2. Target domain enter karein (masalan: api.example.com)
  3. Timeline dekh kar mukhtalif dates select karein
  4. Archived snapshots ko analyze karein
  5. Sensitive information ya purane endpoints dhoondhein

Automation Ke Liye Tools

Manual searching ke ilawa, automated tools bhi available hain:

  • waybackurls - Command line tool jo URLs extract karta hai
  • gau (GetAllUrls) - Multiple sources se URLs fetch karta hai
  • waybackpack - Complete website download karne ke liye

üéØKhaas Tawajjo Dene Wali Cheezein

Jab Wayback Machine istemal kar rahe hon, to in specific targets par focus karein:

1. robots.txt Files

Yeh files aksar directories aur files ki list deti hain jinhe web crawlers ko index nahi karna chahiye. Kabhi kabhi, yeh "disallowed" paths chhupi hui API endpoints ya administrative interfaces ko zahir kar sakte hain.

üí° robots.txt Example:

Disallow: /admin/ Disallow: /api/v1/internal/ Disallow: /backup/ Disallow: /dev-api/
2. JavaScript Files

In mein aksar hardcoded API endpoints, API keys, ya aisa logic ho sakta hai jo yeh zahir karta hai ke API front end ke saath kaise interact karta hai.

üí° JS Files Mein Dhoondhein:

- API endpoints URLs - Authentication tokens - API keys aur secrets - Internal function names
3. Swagger/OpenAPI Documentation

Agar site ki public API documentation pehle thi, toh Wayback Machine ne use archive kar liya hoga, jo available endpoints aur unke mutawaqqo parameters ke bare mein bohat si maloomat faraham karta hai.

üí° Documentation Files:

/swagger.json /api-docs /openapi.yaml /docs/api
4. Developer Blogs Ya Forums

Archived versions mein aisi discussions ya posts mil sakti hain jo ghalti se API ke andaruni kaamon ke bare mein tafseelat zahir kar dain.

5. Configuration Files

Purane configuration files mein database connections, API keys, aur doosri sensitive information ho sakti hai.

üí° Common Config Files:

.env config.json settings.xml database.yml

üìäPractical Example - Step by Step

Scenario: API Endpoints Dhoondhna

  1. Target identify karein: Maan lo aap api.example.com ko analyze kar rahe hain
  2. Wayback Machine par search karein: https://web.archive.org par ja kar URL enter karein
  3. Timeline explore karein: 2-3 saal purane snapshots dekhein
  4. Documentation dhoondhein: /docs, /api-docs paths check karein
  5. JavaScript files analyze karein: Purani JS files mein endpoints search karein
  6. robots.txt check karein: Disallowed paths dekhein
  7. Findings document karein: Saari discovered endpoints ki list banayein
  8. Current site par test karein: Purane endpoints ko current API par try karein

üí° Ahem Nuqta

Wayback Machine ka istemal karna waqt mein peechey ja kar target ki web presence ke bare mein tareekhi data jama karne jaisa hai, jo aksar bhool gaye ya chhupi hui API attack surfaces ko saamne la sakta hai. Yeh passive reconnaissance ka ek taqatwar tareeqa hai jo kisi bhi direct interaction ke bagair valuable information provide karta hai.

‚ö†Ô∏èLimitations Aur Considerations

  • Incomplete Archives

    Har page ya har date archived nahi hota. Important content miss ho sakta hai.

  • Dynamic Content

    JavaScript-heavy sites ya dynamic content properly archive nahi hota.

  • Authentication Required Pages

    Login ke peeche ki content Wayback Machine archive nahi kar sakti.

  • robots.txt Respect

    Wayback Machine robots.txt ko respect karta hai, to kuch content archive hi nahi hota.

üìå Khulasa

Wayback Machine information gathering phase ka ek unique aur powerful tool hai. Yeh aapko time machine ki tarah kaam karne deta hai - aap dekh sakte hain ke target website ya API pehle kaisa tha, kya expose tha, aur developers ne kya mistakes ki thin.

Purane documentation, deprecated endpoints, accidentally exposed secrets, aur historical changes ko analyze kar ke aap aise vulnerabilities dhoondh sakte hain jo current analysis se nazar nahi aate. Yeh passive reconnaissance technique hai jo bina kisi direct interaction ke valuable intelligence provide karti hai.
