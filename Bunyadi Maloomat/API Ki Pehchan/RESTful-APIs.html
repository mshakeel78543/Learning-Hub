RESTful APIs web service communication ka ek khaas andaaz (style) hai jo Representational State Transfer se liya gaya hai. Yeh koi protocol nahi, balkay kuch architectural usoolon (principles) ka majmooa hai jo web services ko design karne mein rehnumai karte hain. In usoolon ka maqsad web services ko halka-phulka (lightweight), qabil-e-dekhbhal (maintainable), aur barhanay ke qabil (scalable) banana hai.

üìã RESTful APIs Ke Bunyadi Usool

1. Resource-Based

Wasail Par Mabni

Ek RESTful API mein, har cheez ko ek resource samjha jata hai. Resource koi bhi data ho sakta hai jise naam diya ja sake, jaise ek user, ek product, ek order, ya ek document.


Har resource ki pehchan ek munfarid URL (Uniform Resource Locator) se hoti hai.

/users ‚Üí Sab users /users/123 ‚Üí User ID 123 /products ‚Üí Sab products /orders/456 ‚Üí Order ID 456

2. Stateless

Bila-Haalat

Client se server tak har request mein woh saari maloomat honi chahiye jo request ko samajhne ke liye zaroori hain. Server ko requests ke darmiyan client ka koi context store nahi karna chahiye.


Iska matlab hai ke server kisi khaas client ke saath pichli baatcheet ko "yaad" nahi rakhta. Har request azaad hoti hai.


Faiday: Scalability behtar hoti hai kyunki koi bhi server koi bhi request handle kar sakta hai, aur yeh system ko mazboot banata hai.

3. Client-Server Architecture

Client-Server Daanchey

Client (maslan, web browser, mobile app, ya doosra server) aur server ke darmiyan saaf alag-alag (separation) hoti hai.


Clients: Sirf user interface aur user experience se mutalliq hote hain.


Servers: Data storage aur processing ka intizam karte hain.


Yeh alag-alag unhein azaadana taur par taraqqi karne ki ijazat deti hai.

4. Uniform Interface

Yaksaa Interface

Yeh ek aham pabandi hai jo poore system architecture ko aasaan banati hai.


Is mein kai choti pabandiyan shamil hain:

  • Identification of Resources: Resources ko URIs se pehchana jata hai
  • Manipulation Through Representations: Resources ki numaindagiyon ka tabadala
  • Self-Descriptive Messages: Har paigham khud-bayan karda hota hai
  • HATEOAS: Hypermedia as the Engine of Application State

Uniform Interface Ki Tafseel:

üîç Identification of Resources (Wasail ki Pehchan):

Resources ko URIs se pehchana jata hai.

üîÑ Manipulation of Resources Through Representations:

Jab ek client kisi resource se interact karta hai, toh woh us resource ki numaindagiyon (representations) ka tabadala karke karta hai. Misaal ke taur par, agar aap user data lena chahte hain, toh server us user ki JSON ya XML representation bhej sakta hai. Ise update karne ke liye, aap ek tabdeel shuda representation bhejte hain.

üìù Self-Descriptive Messages (Khud-Bayan Karda Paigham):

Har paigham (request ya response) mein itni maloomat honi chahiye jo paigham ko process karne ka tareeqa bayan kar saken. HTTP headers yahan ek aham kirdaar ada karte hain:

  • Content-Type: Server ko batata hai ke data kis format mein hai
  • Accept: Client ko batata hai ke woh kis format ko tarjeeh deta hai

üîó Hypermedia as the Engine of Application State (HATEOAS):

Yeh sab se mushkil aur aksar kam nafiz kiya jane wala usool hai.

Matlab: Responses mein mutalliq (related) resources ke links shamil hone chahiye, jo client ko yeh batate hain ke woh agle kaunse actions le sakta hai.

Misaal: Ek user ke response mein us user ko update karne ya uske orders dekhne ka link shamil ho sakta hai.

üîß Standard HTTP Methods Ka Istemal

RESTful APIs resources par operations karne ke liye standard HTTP methods (jinhein verbs bhi kehte hain) ka faida uthate hain:

GET

Hasil Karna

Ek resource ya resources ke collection ko hasil karna.

GET /users ‚Üí Sab users hasil karna GET /users/123 ‚Üí User 123 hasil karna
POST

Banana

Ek naya resource banana.

POST /users ‚Üí Naya user banana
PUT

Update Karna

Ek maujooda resource ko update karna, aam taur par poore resource ko naye data se badal dena.

PUT /users/123 ‚Üí User 123 ko update karna
PATCH

Juzi Update

Ek maujooda resource ko juzi (partially) update karna.

PATCH /users/123 ‚Üí User 123 ke sirf ek khaas field ko update
DELETE

Hatana

Ek resource ko hatana.

DELETE /users/123 ‚Üí User 123 ko delete karna

üîí API Hacking Ke Hawalay Se

‚ö†Ô∏è Ahmiyat:

RESTful APIs ko samajhna buniyadi hai kyunki zyada tar modern web applications inhi par inhisaar karte hain.

Jab aap API hacking kar rahe honge, toh aap aksar RESTful endpoints se interact kar rahe honge. HTTP methods, status codes, request/response structures, aur resources ki pehchan kaise hoti hai, iska ilm inke liye zaroori hai:

üîç Maloomat Jama Karna

üìù Mukhtasar

RESTful APIs - Key Takeaways:

  • Architectural Style: Protocol nahi, balkay principles ka majmooa
  • Resource-Based: Har cheez ek resource hai (URL se pehchana jata hai)
  • Stateless: Har request azaad aur mukammal hoti hai
  • Standard HTTP Methods: GET, POST, PUT, PATCH, DELETE
  • Flexible: JSON, XML, ya doosre formats support karta hai
  • Modern Web Ka Foundation: Zyada tar modern applications RESTful APIs istemal karti hain
  • API Hacking Ke Liye Zaroori: Endpoints, methods, aur structures ko samajhna critical hai
