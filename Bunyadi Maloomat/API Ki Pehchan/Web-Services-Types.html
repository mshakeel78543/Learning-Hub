1. RESTful APIs

Representational State Transfer APIs

Yeh Kya Hai:

Aaj kal web services architecture ki sab se aam qism hai. RESTful APIs stateless hote hain, matlab client se server tak har request mein woh tamam maloomat shamil hoti hain jo request ko samajhne ke liye zaroori hain. Yeh resources par operations karne ke liye standard HTTP methods (GET, POST, PUT, DELETE) istemal karte hain, jinhein URLs se pehchana jata hai.

Khas Khususiyaat:

  • Stateless: Requests ke darmiyan server par koi session maloomat store nahi hoti.
  • Client-Server: Client aur server ke darmiyan wazeh tafreeq.
  • Cacheable: Performance behtar banane ke liye responses ko cache kiya ja sakta hai.
  • Layered System: Intermediaries jaise load balancers aur proxies shamil ho sakte hain.
  • Uniform Interface: Standard HTTP methods aur resource URIs istemal karta hai.

üîí Hacking Relevance:

REST ko samajhna endpoints, HTTP methods, aur authentication se mutaliq mumkina vulnerabilities ki nishandahi ke liye zaroori hai. Khaas taur par:

  • Access Control: IDOR (Insecure Direct Object References) attacks
  • Injection Attacks: Requests ke parameters ya body mein malicious data
  • Authentication Bypass: Token aur session management issues
  • Rate Limiting: API abuse aur DoS attacks

2. SOAP APIs

Simple Object Access Protocol APIs

Yeh Kya Hai:

SOAP web services ki implementation mein structured information ka tabadla karne ke liye aik protocol hai. Yeh message formatting ke liye XML par bohat zyada inhisaar karta hai aur aam taur par HTTP par kaam karta hai, lekin SMTP ya TCP jaise deegar protocols bhi istemal kar sakta hai.

Khas Khususiyaat:

  • Protocol-based: Yeh aik rasmi protocol hai jis ke sakht qawanin hain.
  • XML-based: Tamam messages XML format mein hote hain.
  • WSDL: (Web Services Description Language) Service ke operations, parameters, aur data types ko bayan karti hai.
  • Stateful (ho sakta hai): Khaas extensions ke zariye stateful operations ko support kar sakta hai.
  • Built-in Security: WS-Security standards support karta hai.

üîí Hacking Relevance:

Hamlawar WSDL files talash karte hain taake API ki structure ko samajh saken. SOAP APIs mein khaas vulnerabilities:

  • XML External Entity (XXE) Injection: XML par inhisaar ki wajah se zyada common
  • XML Injection: Malicious XML data injection
  • WSDL Enumeration: API structure aur endpoints ki discovery
  • Header Manipulation: SOAP headers mein authentication bypass
  • Message Tampering: XML message structure manipulation

3. GraphQL APIs

Query Language for APIs

Yeh Kya Hai:

GraphQL APIs ke liye aik query language hai aur un queries ko aap ke mojooda data se poora karne ke liye aik runtime hai. Yeh clients ko bilkul wohi data talab karne ki ijazat deta hai jo unhein chahiye, na zyada na kam, sirf aik endpoint par aik single query bhej kar.

Khas Khususiyaat:

  • Single Endpoint: Aam taur par, tamam queries aur mutations aik hi /graphql endpoint par jati hain.
  • Client-driven Data Fetching: Clients batate hain ke woh kya data chahte hain.
  • Strongly Typed Schema: API ki salahiyaten aik schema se bayan ki jati hain.
  • Introspection: Clients schema ko khud query kar sakte hain taake woh samajh saken ke kya data available hai.
  • Flexible Queries: Multiple resources ko aik hi request mein fetch karna.

üîí Hacking Relevance:

GraphQL naye attack vectors paida karta hai jo traditional REST APIs mein nahi paaye jate:

  • Introspection Queries: API ki structure ke bare mein sensitive maloomat leak kar sakti hain
  • Batching Attacks: Multiple queries ko aik request mein bhej kar abuse
  • Alias Abuse: Same field ko multiple baar query karna
  • Resource Exhaustion: Deeply nested queries se DoS attacks
  • Authorization Issues: Field-level access control bypass

üîç 1. Technology Ki Pehchan

Jab aap kisi API ka saamna karte hain, to aap uske rawaiye, endpoints, aur data formats ki bunyad par uski zereen technology (REST, SOAP, GraphQL) ka andaza laga sakte hain.

  • Multiple endpoints aur HTTP methods ‚Üí Likely REST
  • XML-based messages aur WSDL file ‚Üí SOAP
  • Single /graphql endpoint ‚Üí GraphQL

üéØ 2. Vulnerabilities Ka Andaza

  • REST ‚Üí IDOR, Rate Limiting, Token Issues
  • SOAP ‚Üí XXE, XML Injection, WSDL Exposure
  • GraphQL ‚Üí Introspection, Batching, Nested Queries
