üìñ HTTP Status Codes Kya Hain?

Definition:

Jab aapka browser ya koi API client server ko ek request bhejta hai, to server us request ko process karta hai aur ek response wapas bhejta hai.


Is response ka ek hissa status code hota hai, jo adad mein us request ka nateeja batata hai.


Format: Teen-digit number (100-599)

HTTP/1.1 200 OK Content-Type: application/json { "message": "Success" }

‚Üë 200 = Status Code

üìö Detailed Status Codes

1xx - Informational Responses

‚ÑπÔ∏è

Yeh batate hain ke request receive ho gayi thi aur samajh li gayi thi. Yeh aarzui responses hote hain, jo progress batate hain.

100
Continue
Client ko request jari rakhni chahiye. Server ne request headers receive kar liye hain aur client ko request body bhejni chahiye.
101
Switching Protocols
Server client ke Upgrade header request ko accept kar raha hai aur protocols switch kar raha hai.

üí° Note: 1xx codes aam API interactions mein foran nateejon ke liye kam istemal hote hain. Yeh mostly WebSocket connections ya streaming scenarios mein dekhe jate hain.

2xx - Success Responses

‚úÖ

Yeh batate hain ke request kamyabi se receive ho gayi, samajh li gayi, aur qabool kar li gayi.

200
OK
Sab se aam status code. Request kamyab thi, aur server mangi hui data wapas bhej raha hai. GET requests ke liye standard response.
201
Created
Request kamyab thi, aur nateeje ke taur par ek naya resource banaya gaya. POST requests ke baad common.
204
No Content
Server ne request ko kamyabi se process kar liya, lekin wapas bhejne ke liye koi content nahi hai. DELETE requests ke liye useful.
202
Accepted
Request processing ke liye accept kar li gayi hai, lekin processing abhi complete nahi hui. Async operations ke liye.

3xx - Redirection Messages

‚Ü™Ô∏è

Yeh batate hain ke request ko mukammal karne ke liye client ko mazeed action lena hoga, aam taur par kisi doosre URL par redirect karna hota hai.

301
Moved Permanently
Manga hua resource mustaqil taur par naye URL par muntaqil kar diya gaya hai. Future requests naya URL istemal karein.
302
Found
Manga hua resource aarzi taur par ek mukhtalif URL par maujood hai. Client ko original URL istemal karna chahiye.
304
Not Modified
Resource modify nahi hua hai. Client cached version istemal kar sakta hai. Caching ke liye useful.
307
Temporary Redirect
302 ki tarah, lekin guarantee hai ke HTTP method change nahi hoga redirect ke baad.

4xx - Client Error Responses

‚ö†Ô∏è

Yeh batate hain ke client ki request mein ghalti thi. Iska matlab hai ke client ne kuch ghalat kiya.

400
Bad Request
Server client ki ghalti ki wajah se request process nahi kar sakta, masalan, ghalat syntax ya invalid parameters.
401
Unauthorized
Client ko mangi hui response hasil karne ke liye authenticate karna hoga. Login required!
403
Forbidden
Client ko content tak rasai ka haq nahi hai. Authentication ho sakti hai, lekin permissions nahi hain.
404
Not Found
Sab se mashhoor error! Server ko manga hua resource nahi mila. URL ghalat hai ya resource exist nahi karta.
405
Method Not Allowed
Request method us resource ke liye supported nahi hai. Masalan, DELETE allowed nahi hai.
429
Too Many Requests
User ne ek muqarrar waqt mein bohat zyada requests bhej di hain. Rate limiting lag gayi hai!

5xx - Server Error Responses

‚ùå

Yeh batate hain ke server ek sahi request ko poora karne mein nakam raha. Masla server ki taraf hai!

500
Internal Server Error
Ek aam ghalti ka paigham, jo us waqt diya jata hai jab koi ghair mutawaqqa soorat e haal paish aaye.
501
Not Implemented
Server request method ko support nahi karta ya usay fulfill nahi kar sakta.
502
Bad Gateway
Server, gateway ya proxy ke taur par kaam karte waqt, invalid response mili.
503
Service Unavailable
Server request ko handle karne ke liye tayar nahi hai, aksar maintenance ya overload hone ki wajah se.
504
Gateway Timeout
Server, gateway ke taur par kaam karte waqt, upstream server se timely response nahi mila.

üîí API Hacking Mein Ahmiyat

Security Perspective

API Hacking ke liye, status codes ko samajhna kai wajuhat ki bina par bohat zaroori hai:

üîç

1. Maloomat Jama Karna

Attackers API ke structure aur vulnerabilities ke bare mein seekhte hain:

  • 401 Unauthorized: Authentication Bypass ki koshish
  • 403 Forbidden: Broken Access Control testing
  • 404 Not Found: Endpoint enumeration
  • 405 Method Not Allowed: Allowed methods discover
  • Rate Limiting: 429 dekh kar bypass techniques
üêõ

2. Hamlon Ki Debugging

Status codes attacker ko payload ka nateeja samajhne mein madad karte hain:

  • 400 Bad Request: Payload format ghalat hai
  • 500 Internal Server Error: Payload ne error trigger kiya
  • 200 OK: Injection successful ho sakta hai
  • Response time: Blind injection detection
üí•

3. Vulnerabilities Ki Pehchan

Specific vulnerabilities ke indicators:

  • Information Disclosure: 5xx errors mein sensitive data
  • User Enumeration: Valid vs invalid users ke liye mukhtalif codes
  • IDOR Testing: 200 vs 404 patterns
  • SQL Injection: 500 errors se database hints
  • XSS Detection: Response codes aur content analysis

4. Automated Tools

Security tools status codes ka analysis karte hain:

  • Burp Suite: Status code-based filtering
  • OWASP ZAP: Automated scanning patterns
  • Postman: Test assertions on status codes
  • Custom Scripts: Status code monitoring
  • Fuzzing Tools: Response pattern analysis
