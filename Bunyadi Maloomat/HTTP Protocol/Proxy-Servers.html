API Hacking ‚Üí Bunyadi Maloomat ‚Üí HTTP Protocol ‚Üí Proxy Servers

Ek proxy server client ki taraf se aane wali requests ke liye ek intermediary ka kaam karta hai, jo doosre servers se resources talash kar raha hota hai.

Client seedha destination server se connect hone ke bajaye proxy server se connect hota hai, aur phir proxy server us request ko destination tak forward karta hai.

Yeh position unhe network traffic ko rokne (intercept), muaina karne (inspect), tabdeel karne (modify), aur aage bhejne (forward) ki ijazat deti hai.

üìñ Proxy Server Kya Hai?

Definition:

Ek proxy server ek intermediary server hai jo client device (jaise aapka computer ya phone) aur Internet ke darmiyan requests ko route karta hai.


Yeh aapke browser/tool (client) aur API server (destination) ke darmiyan baitha hota hai, jo complete control provide karta hai network traffic par.

  • Response Patterns: Success/failure indicators
  • Hidden Fields: Debug information

  • WAF Bypass: Web Application Firewall se bachna

‚öîÔ∏è Common Attack Vectors Through Proxies

Proxy Se Possible Attacks

1. Man-in-the-Middle (MITM) Attacks

Proxy ka istemal kar ke communication ko intercept aur modify karna. SSL/TLS certificates ko bypass kar ke encrypted traffic ko decrypt karna.

Attacker Proxy intercepts: Client ‚áÑ Malicious Proxy ‚áÑ Server Both parties think they're communicating securely!
2. Request Tampering

API requests ko modify kar ke malicious payloads inject karna, parameters change karna, ya authentication bypass karna.

Original: GET /api/users/123 Modified: GET /api/users/1' OR '1'='1 ‚Üê SQL Injection via proxy manipulation!
3. Session Hijacking

Proxy se session tokens ya cookies capture kar ke unauthorized access hasil karna.

Captured Cookie: Set-Cookie: sessionid=abc123; Path=/ Attacker uses this to impersonate user!
4. Rate Limiting Bypass

Multiple proxy servers ya rotating IPs istemal kar ke API rate limits ko bypass karna.

Request 1: Proxy IP 1.2.3.4 Request 2: Proxy IP 5.6.7.8 Request 3: Proxy IP 9.10.11.12 ‚Üê Different IPs = Rate limit bypassed!
5. Content Injection

Proxy ke through responses mein malicious content inject karna jaise XSS payloads.

Original Response: {"name": "Ali"} Injected: {"name": "<script>alert(1)</script>"} ‚Üê XSS payload injected via proxy!
